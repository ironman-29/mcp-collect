GitHub 개인 액세스 토큰(PAT) 설정 가이드

Claude MCP 설정에서 GitHub 연동을 위해 GITHUB_PERSONAL_ACCESS_TOKEN을 설정하는 방법을 안내합니다. 이 가이드를 통해 GitHub Personal Access Token(PAT)을 생성하고 설정 파일에 적용하는 방법을 알아보세요.

1 GitHub 로그인

먼저 GitHub 계정에 로그인합니다.

https://github.com

2 개인 토큰 생성 페이지로 이동

다음 경로를 통해 토큰 생성 페이지로 이동합니다:

  1. 우측 상단 프로필 > Settings
  2. 왼쪽 메뉴에서 Developer settings 선택
  3. Personal access tokens
  4. Tokens (classic)
  5. Generate new token (classic) 클릭

또는 다음 링크로 직접 접속할 수 있습니다:

https://github.com/settings/tokens

3 토큰 생성

  • Note: 예) Claude MCP Token
  • Expiration: 권한 유지 기간 설정 (예: 90일, 혹은 No expiration)
  • Scopes(권한): Claude가 GitHub 리포지토리에 접근하려면 최소 다음 권한이 필요합니다:
    • repo (필수)
    • read:user (옵션)
    • workflow (GitHub Actions 사용 시)
  • 설정 완료 후 Generate token 클릭
🔐 주의: 생성 직후 한 번만 볼 수 있으므로 반드시 복사해두세요!

4 MCP 설정 파일에 토큰 적용

아래처럼 claude_desktop_config.json 또는 해당 MCP 설정 JSON에 붙여넣기 합니다:

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-github"
      ],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
      }
    }
  }
}
  • ghp_로 시작하는 GitHub 토큰을 ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxx 자리에 넣습니다.
  • JSON에서 큰따옴표(")와 쉼표 위치를 유의하세요.
  • 설정 후 Claude Desktop을 재시작해야 합니다.

Claude에서 GitHub 활용 예시

"내 GitHub 리포지토리 리스트를 보여줘."
"사용자 이름 'octocat'의 공개 저장소를 검색해줘."
"이 리포지토리의 `README.md`를 요약해줘."

추가 팁

만약 보안을 위해 토큰을 별도 .env 파일에 관리하고 싶다면, Node.js 기반 서버로 확장해서 .env 환경에서 불러오는 방식도 가능합니다.

보안 주의: 토큰 누출 방지를 위해 GitHub 토큰은 절대 공개 저장소에 올리지 마세요.
